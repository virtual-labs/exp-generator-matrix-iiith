Generator Matrix

In Experiment-1, we provided a brief introduction to block codes and also defined a class of block codes, linear block codes. For rest of the experiments in this virtual lab we shall now focus on linear block codes. In Experiment-1, we also introduced two simple classes of linear block codes, repetition and single parity check codes. In this experiment, we shall focus on the theory general linear block codes. We shall study two important matrices associated with the given linear block code, generator matrix (denoted by GGG) and parity check matrix (denoted by HHH).

The theory associated with Experiment-2 is divided into two parts:
(1) Generator Matrix
(2) Parity Check Matrix

1 Generator matrix

1.1 Obtaining a generator matrix from the given codebook

In Examples-3 and 4, we illustrated how to obtain linear block code corresponding to the given generator matrix. Let us now discuss how to obtain a generator matrix from the given linear block code C(n,k)\mathcal{C}(n,k)C(n,k). Recall that a linear block code C(n,k)\mathcal{C}(n,k)C(n,k) is a kkk-dimensional subspace of the vector space F2n\mathbb{F}_2^nF2n​. Let g1,g2,…,gk\mathbf{g}_1, \mathbf{g}_2, \ldots, \mathbf{g}_kg1​,g2​,…,gk​ be a basis of this subspace (preliminaries about basis of a vector space are discussed in Experiment-1, Part-1 Theory). Obtain a matrix G∈F2k×nG \in \mathbb{F}_2^{k \times n}G∈F2k×n​ corresponding to this basis as follows

G=[←g1→←g2→⋮←gk→].\begin{align} G = \begin{bmatrix} \leftarrow & \mathbf{g}_1 & \rightarrow \\ \leftarrow & \mathbf{g}_2 & \rightarrow \\ & \vdots & \\ \leftarrow & \mathbf{g}_k & \rightarrow \end{bmatrix}. \end{align}G=​←←←​g1​g2​⋮gk​​→→→​​.​​

Note that the given code C(n,k)=span{g1,g2,…,gk}\mathcal{C}(n,k)=\hspace{0.01in}{span}\{\mathbf{g}_1, \mathbf{g}_2, \ldots, \mathbf{g}_k\}C(n,k)=span{g1​,g2​,…,gk​}, i.e., the rows of the matrix GGG defined in Eq. (1) generate (span) the linear block code C(n,k)\mathcal{C}(n,k)C(n,k). Hence this matrix GGG is termed as a generator matrix. Note that different bases of the vector space C(n,k)\mathcal{C}(n,k)C(n,k) will correspond to a different generator matrix and hence a generator matrix for the given code is not unique.
For Examples-3 and 4 it can be verified that the rows corresponding to the respective generator matrices does form a basis for the respective linear block codes.

1.2 Systematic generator matrix

A linear block code C(n,k)\mathcal{C}(n,k)C(n,k) is said to be systematic if every codeword v∈C(n,k)\mathbf{v} \in \mathcal{C}(n,k)v∈C(n,k) can be written as v=[mp],\begin{align*} \mathbf{v} = \begin{bmatrix} \mathbf{m} & \mathbf{p} \end{bmatrix}, \end{align*}v=[m​p​],​ where m∈F2k\mathbf{m} \in \mathbb{F}_2^km∈F2k​ is the message corresponding to the codeword v\mathbf{v}v and p∈F2n−k\mathbf{p} \in \mathbb{F}_2^{n-k}p∈F2n−k​ is the vector corresponding to parity bits. It can be seen that Examples-1, 2, and 3 are systematic codes whereas Example-4 is not a systematic code.

For a systematic code, it is possible to obtain a generator matrix that can be written as G=[IkP],\begin{align*} G = \begin{bmatrix} I_k & P \end{bmatrix}, \end{align*}G=[Ik​​P​],​ where PPP is some matrix in F2k×n−k\mathbb{F}_2^{k \times n-k}F2k×n−k​ and Ik∈F2k×kI_k \in \mathbb{F}_2^{k \times k}Ik​∈F2k×k​ is an identity matrix given by

Ik=[10…001…0⋮⋮…⋮00…1].\begin{align*} I_k = \begin{bmatrix} 1 & 0 & \ldots & 0 \\ 0 & 1 & \ldots & 0 \\ \vdots & \vdots & \ldots & \vdots \\ 0 & 0 & \ldots & 1 \end{bmatrix}. \end{align*}Ik​=​10⋮0​01⋮0​…………​00⋮1​​.​ Note that an identity matrix IkI_kIk​ is a diagonal matrix with 111 as diagonal entries.

2 Parity check matrix

A linear block code can also be defined using a parity check matrix H∈F2n−k×nH\in \mathbb{F}_2^{n-k \times n}H∈F2n−k×n​ as follows.

Definition 1 Linear block code C(n,k)\mathcal{C}(n,k)C(n,k)): For the given parity check matrix H∈F2n−k×nH\in \mathbb{F}_2^{n-k \times n}H∈F2n−k×n​, the linear block corresponding to it is defined as the collection of all possible vectors v∈F2n\mathbf{v} \in \mathbb{F}_2^nv∈F2n​ that satisfy the condition vHT=0\mathbf{v}H^T = \mathbf{0}vHT=0, i.e., C(n,k)={v∈F2nsuchthatvHT=0}.\begin{align*} \mathcal{C}(n,k) = \left\{ \mathbf{v} \in \mathbb{F}_2^n \hspace{0.05in}{such} \hspace{0.05in}{} {that}\hspace{0.05in}{} \mathbf{v}H^T = \mathbf{0} \right\}. \end{align*}C(n,k)={v∈F2n​suchthatvHT=0}.​

It can be seen from Definition 1 that, one can verify whether the given vector is a codeword or not using a parity check matrix. Similar to generator matrix, a parity check matrix also need not be unique. We shall next define the dual code C⊥(n,n−k)\mathcal{C}^{\perp}(n,n-k)C⊥(n,n−k) of the linear block code C(n,k)\mathcal{C}(n,k)C(n,k).


Definition 2 Dual code: Consider a linear block code C(n,k)\mathcal{C}(n,k)C(n,k) with parity check matrix HHH. Let C⊥(n,n−k)\mathcal{C}^{\perp}(n,n-k)C⊥(n,n−k) be the linear block obtained by choosing HHH as its generator matrix. Then C⊥(n,n−k)\mathcal{C}^{\perp}(n,n-k)C⊥(n,n−k) is said to be the dual code of C(n,k)\mathcal{C}(n,k)C(n,k), i.e., for any v∈C⊥(n,k)\mathbf{v} \in \mathcal{C}^{\perp}(n,k)v∈C⊥(n,k) and w∈C⊥(n,n−k)\mathbf{w} \in \mathcal{C}^{\perp}(n,n-k)w∈C⊥(n,n−k) we have vwT=0\mathbf{v}\mathbf{w}^T = 0vwT=0.

We shall now summarize some properties of parity check matrices.

  • A generator matrix G∈F2k×nG \in \mathbb{F}_2^{k \times n}G∈F2k×n​ and parity check matrix H∈F2n−k×nH\in \mathbb{F}_2^{n-k \times n}H∈F2n−k×n​ associated with the given linear block code satisfy the following condition GHT=0.\begin{align*} G H^T = \mathbf{0}. \end{align*}GHT=0.​
  • Consider a linear block code C(n,k)\mathcal{C}(n,k)C(n,k) with parity check matrix HHH. Then a vector v∈F2n\mathbf{v} \in \mathbb{F}_2^nv∈F2n​ is a codeword in C(n,k)\mathcal{C}(n,k)C(n,k) if and only if vHT=0\mathbf{v}H^T = \mathbf{0}vHT=0.
  • Consider a linear block code C(n,k)\mathcal{C}(n,k)C(n,k) with parity check matrix HHH. Then the minimum distance dmin⁡d_{\min}dmin​ (see Experiment-1 Theory, Definition 13) of C(n,k)\mathcal{C}(n,k)C(n,k) is equal to the smallest number of columns of HHH that sum to 0\mathbf{0}0. It can be verified using this property that the minimum distance dmin⁡d_{\min}dmin​ of code in Example-3 is equal to 222 (see Eq. (3)).
